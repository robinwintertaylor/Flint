# Flint E2E System Runbook — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single ordered end-to-end runbook (`docs/e2e-runbook.md`) and companion automated test file (`dashboard/tests/e2e.test.js`) that verify every feature area of the live Flint stack, then execute them.

**Architecture:** Two artifacts — a human/agent-readable runbook document covering all 20 sections in dependency order, and a Node.js live-stack test file that automates the API-testable sections. Browser UI sections are executed via browser automation. CLI sections via terminal. All test data is prefixed `e2e-test-` and cleaned up after each section.

**Tech Stack:** Node.js ESM, node:test, node:assert/strict, fetch (built-in), browser automation via mcp__claude-in-chrome tools

## Global Constraints

- Dashboard runs at `http://localhost:3000`
- Router runs at `http://localhost:3001`
- Forgejo runs at `http://localhost:3030`
- Ollama runs at `http://localhost:11434`
- All test data prefixed `e2e-test-` — never touch existing data
- Sections gated on `E2E_FULL=1` env var: agent spawn (S5), Forgejo PR flow (S16)
- Telegram / LM Studio / GitHub sections are skipped — services not configured
- `node --test dashboard/tests/e2e.test.js` must pass with zero failures on the non-gated sections

/**
 * Flint E2E Test Suite — hits the live running stack (no mocks).
 * Requires: dashboard at :3000, router at :3001
 * Full mode: E2E_FULL=1 node --test dashboard/tests/e2e.test.js

import { test, before } from 'node:test';
import assert from 'node:assert/strict';

const BASE  = 'http://localhost:3000';
const ROUTER = 'http://localhost:3001';
const FULL  = process.env.E2E_FULL === '1';

async function api(method, path, body, base = BASE) {
  const opts = { method, headers: { 'Content-Type': 'application/json' } };
  if (body !== undefined) opts.body = JSON.stringify(body);
  const res = await fetch(`${base}${path}`, opts);
  return res;
}

// ── Preflight ──────────────────────────────────────────────────────
before(async () => {
  const res = await fetch(`${BASE}/health`).catch(() => null);
  if (!res?.ok) throw new Error(`Dashboard not reachable at ${BASE} — start PM2 before running E2E tests`);
  const res2 = await fetch(`${ROUTER}/health`).catch(() => null);
  if (!res2?.ok) throw new Error(`Router not reachable at ${ROUTER} — start PM2 before running E2E tests`);
});

// ── S1: Health ─────────────────────────────────────────────────────
test('[S1] GET /health returns ok with db connected', async () => {
  const r = await api('GET', '/health');
  assert.equal(r.status, 200);
  const b = await r.json();
  assert.equal(b.status, 'ok');
  assert.equal(b.db, 'connected');
  assert.ok('forgejo' in b, 'forgejo field missing');
  assert.ok('ollama'  in b, 'ollama field missing');
});

test('[S1] GET router /health returns ok', async () => {
  const r = await fetch(`${ROUTER}/health`);

});

test('[S1] Forgejo reachable (health says reachable)', async () => {
