# Flint — SP6: Multi-Agent Isolation & Production Hardening
**Sub-project 6 of 6**
**Date:** 2026-06-24
**Status:** Approved

---

## 1. Problem

Agents running on isolated worktrees (SP5) currently merge directly back to `master` with no review step. There is also no process resilience — if the dashboard crashes it stays down until manually restarted, and there is no structured logging to diagnose failures.

---

## 2. Scope

**In:**
- Forgejo (self-hosted Git UI) via Docker Compose — local PR review for agent worktree branches
- Automatic PR creation when an agent exits with an active worktree
- PR status polling (open → merged/closed) surfaced in the dashboard
- PM2 process management — crash restart + boot persistence for dashboard and router
- Structured JSON logging via a thin `logger.js` module
- `GET /health` endpoint on the dashboard
- `scripts/forgejo-init.ps1` bootstrap script

**Out (noted for later):**
- Gitea Actions CI runner (Option C) — add `gitea-runner` to `docker-compose.yml` + branch protection rules; no code changes required
- External remote push (GitHub, GitLab)
- Dashboard authentication / HTTPS
- Budget alerts, cost forecasting

## 5. Data Flow — PR Lifecycle

```
Agent exits with active worktree
  → terminal.js: broadcast worktree_pending (existing)
  → server.js catches worktree_pending event
  → forgejo.pushBranch(branch)          git push forgejo <branch>
  → forgejo.createPR(branch, agentName) POST /api/v1/repos/robin/flint/pulls
  → db.setAgentPR(name, prNumber, prUrl, 'open')
  → broadcastToAgent: { type: 'worktree_pr', agent, prUrl, prNumber }
  → UI: Merge/Discard buttons → "View PR" link + open badge

Every 30s (server-side timer):
  → forgejo.getPRStatus(prNumber)       GET /api/v1/repos/robin/flint/pulls/:number
  → if status changed: db.setAgentPR(name, ..., newStatus)
  → broadcastToAgent: { type: 'pr_status', agent, status }
  → if merged: clearAgentWorktree(name)
  → UI: badge updates colour
```

## 6. Forgejo Setup

### docker-compose.yml

```yaml
services:
  forgejo:
    image: codeberg.org/forgejo/forgejo:7
    container_name: flint-forgejo
    ports:
      - "3030:3000"
    volumes:
      - forgejo-data:/data
    environment:
      - USER_UID=1000
      - USER_GID=1000
    restart: unless-stopped

volumes:
  forgejo-data:
```

### Bootstrap script (`scripts/forgejo-init.ps1`)

Run once after `docker compose up -d`:
1. Wait for Forgejo to be ready (`GET /api/v1/version`)
2. Create admin user `robin` via Forgejo API
3. Create repo `flint` (private, no auto-init)
4. Add `forgejo` remote: `git remote add forgejo http://robin:<token>@localhost:3030/robin/flint.git`
5. Push `master` branch to seed the repo
6. Write the API token to `forgejo.token` file at Flint root (git-ignored)

### For Option C (CI) later

Add to `docker-compose.yml`:
```yaml
  gitea-runner:
    image: gitea/act_runner:latest
    ...
```
Enable branch protection via Forgejo API. No changes to application code required.

## 7. `dashboard/forgejo.js`

**Interfaces:**
- `isForgejReachable()` → `boolean` — cheap version check, 2s timeout
- `pushBranch(branch)` → `void` — `git push forgejo <branch>`
- `createPR(branch, agentName)` → `{ prNumber, prUrl }` — POST to Forgejo API
- `getPRStatus(prNumber)` → `'open' | 'merged' | 'closed'`
- `listOpenPRs()` → `[{ prNumber, agentName, branch }]` — used by 30s poll timer

Config read from environment / `forgejo.token` file:
- `FORGEJO_URL` (default `http://localhost:3030`)
- `FORGEJO_TOKEN` (read from `forgejo.token` if env not set)
- `FORGEJO_OWNER` (default `robin`)
- `FORGEJO_REPO` (default `flint`)

## 9. Modified Modules

### `dashboard/terminal.js`

`spawnAgent` gains an optional fourth argument `{ onWorktreePending }`. After the existing `worktree_pending` broadcast in `ptyProcess.onExit`, call the callback:

```js
export function spawnAgent(name, workdir, model, { onWorktreePending } = {}) {
  // ...existing spawn logic...
  ptyProcess.onExit(() => {
    const worktree = getAgentWorktree(name);
    if (worktree?.worktree_branch) {
      broadcastToAgent(name, { type: 'worktree_pending', agent: name, branch: worktree.worktree_branch });
      onWorktreePending?.(name, worktree.worktree_branch);
    }
    // ...existing exit cleanup...
  });
}
```

`createPRForAgent` is a private async function defined in `server.js` (where it has access to `broadcastToAgent`). `terminal.js` does not import it — `server.js` passes it as the `onWorktreePending` callback when calling `spawnAgent`, avoiding a circular dependency.

### `dashboard/worktrees.js`

- Remove `mergeWorktree` export — direct merge is replaced by PR flow
- Keep `createWorktree`, `listWorktrees`, `discardWorktree`
- `discardWorktree` also calls `clearAgentPR(name)` after clearing worktree

### `dashboard/server.js`

- Import `forgejo.js`, `logger.js`
- Add `createPRForAgent(name, branch)` helper: pushBranch → createPR → setAgentPR → broadcast
- Add 30s interval: poll `listOpenPRAgents()` → `getPRStatus()` → update DB + broadcast
- Remove `/worktrees/:agent/merge` route (replaced by PR flow)
- Add `GET /health` route

### `bin/flint.js`

- Remove `flint worktree merge` subcommand (direct merge gone)
- `flint worktree list` now shows `pr_url` and `pr_status` columns

## 12. Structured Logging (`dashboard/logger.js`)

```js
function log(level, msg, data = {}) {
  process.stdout.write(
    JSON.stringify({ ts: new Date().toISOString(), level, msg, ...data }) + '\n'
  );
}
export const info  = (msg, data) => log('info',  msg, data);
export const warn  = (msg, data) => log('warn',  msg, data);
export const error = (msg, data) => log('error', msg, data);
```

Key call sites to migrate from `console.log/error`:
- `server.js`: startup, route errors, PR poll results
- `terminal.js`: agent spawn, exit, cost capture, PR creation errors
- `forgejo.js`: all API call results

## 13. `GET /health`

```json
{
  "status": "ok",
  "uptime": 3600,
  "db": "connected",
  "forgejo": "reachable"
}
```

- `forgejo` field: `"reachable"` or `"unreachable"` (2s timeout on `GET /api/v1/version`)
- `status`: `"ok"` if all connected, `"degraded"` if forgejo unreachable (server keeps running)
- `uptime`: `process.uptime()` in seconds
