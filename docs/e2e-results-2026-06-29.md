## Bugs Found and Fixed

### Bug 1 — PM2 server never binds to port 3000

**Symptom:** PM2 restarting flint-dashboard in a loop; `netstat` shows no listener on :3000; PM2 logs empty.

**Root cause:** PM2 fork mode sets `process.argv[1]` to its own `ProcessContainerFork.js`, not the server script. The ESM `isMain` guard (`process.argv[1] === fileURLToPath(import.meta.url)`) was always false, so the server never called `.listen()`. Node had no active handles and exited immediately, triggering infinite PM2 restart loop.

**Fix:** Added `PM2_HOME !== undefined` as a fallback isMain check. PM2 always sets this env var in managed subprocesses; test shells do not have it, preserving correct test isolation.

**File:** `dashboard/server.js` — commit `a27052b`

---

### Bug 2 — `POST /projects` returns 400 on test re-run

**Symptom:** Second test run fails at S8 project creation with UNIQUE constraint violation.

**Root cause:** `DELETE /projects/:id` is a soft-delete (archives the row). Project names remain UNIQUE-constrained. The hard-coded test project name `e2e-test-project` collided on the second run.

**Fix:** Added `const RUN_ID = Date.now().toString().slice(-6)` and suffixed all test-created project names with the run ID: `e2e-test-project-${RUN_ID}`.

**File:** `dashboard/tests/e2e.test.js` — commit `600d5c3`

---

### Bug 3 — Server crash on `POST /projects/:id/docs` with bad project ID

**Symptom:** Server process crashed mid-test-run with unhandled rejection: `SqliteError: NOT NULL constraint failed: project_docs.project_id`.

**Root cause:** When S8 project creation failed (due to Bug 2), `_projId` was undefined. S14 passed `"undefined"` as the project ID. The `createDoc()` call threw synchronously inside an `async` Express handler. Sync throws inside `async (req, res) => {}` become unhandled promise rejections that crash the Node process.

**Fix:** Wrapped `createDoc()` call in try-catch, returning `500` with the error message on failure.

**File:** `dashboard/server.js` — commit `600d5c3`

## Pre-existing Issue (Not Fixed)

### OpenRouter model IDs outdated in `router/router.json`

**Symptom:** `POST /llm/complete` returns 500; `flint ask` returns `Error: Request failed with status 400`.

**Impact:** S2 LLM complete test skipped; S19 `flint ask` skipped.

**Root cause:** Model slugs in `router.json` (e.g. `mistral/mistral-medium`) are no longer valid OpenRouter IDs. OpenRouter returns 400 for unrecognised model names, which the router re-raises as 500.

**Resolution needed:** Update `router/router.json` with current OpenRouter model slugs. This is a configuration issue, not a code bug.
